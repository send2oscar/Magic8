# 本機 ComfyUI Bridge 架構

## 目的

`XXX` 是固定的 Qwen 服裝替換功能。網站不會直接連入 ComfyUI，也不會公開使用者電腦的 ComfyUI 連接埠。相反地，使用者電腦上的 Bridge 主動透過 HTTPS 向網站領取工作、呼叫僅限本機的 ComfyUI，並將結果回傳至網站。

## 角色與信任邊界

| 元件 | 職責 | 可見資料 |
|---|---|---|
| Dashboard | 由已登入使用者建立 `XXX` 任務、查看安全的進度與結果 | 自己的任務狀態與結果 |
| 網站伺服器 | 驗證照片歸屬、保留／退款點數、簽發短時效來源 URL、保存輸出 | 任務中繼資料與受控儲存存取 |
| 本機 Bridge | 主動領取已租用任務、將來源圖片傳給 `127.0.0.1:8188`、回傳輸出 | 當前任務的來源圖片與處理結果 |
| ComfyUI Desktop | 只執行本機固定 Qwen 工作流程 | Bridge 上傳的當前工作圖片 |

## 任務資料流

```text
使用者點選 XXX
    -> 網站驗證照片與點數，建立持久化任務並保留一點
    -> 已配對 Bridge 輪詢並領取任務租約
    -> 網站提供短時效來源 URL + 固定 Qwen 工作流程規格
    -> Bridge 將圖片上傳至 localhost:8188，僅替換 LoadImage 節點 78 的檔名
    -> Bridge 送出工作流程、輪詢本機 ComfyUI、讀取節點 102 的輸出
    -> Bridge 回報階段並上傳結果
    -> 網站將結果儲存於受控儲存空間、完成歷史紀錄並顯示於圖庫
```

## 配對與憑證

專案擁有者在 Dashboard 產生短時效、一次性配對碼。Bridge 以該配對碼交換隨機產生的裝置憑證；伺服器只保存憑證雜湊。瀏覽器不會取得裝置憑證，且每次 API 呼叫都同時驗證裝置身分與任務租約。重新配對會撤銷舊裝置。

這不是在 ComfyUI 前設定公開 Bearer Token。Bridge 的裝置憑證由配對流程自動產生並保存在使用者自己的電腦，用於防止其他人冒充工作站領取或回傳任務。

## 任務狀態與恢復原則

任務狀態依序為 `queued`、`leased`、`processing`、`completed` 或 `failed`。Bridge 領取任務時取得有限時間的租約；若中斷，租約到期後可安全重新租用或標記失敗。伺服器只在任務尚未終止時退款，避免重複退款。Bridge 離線時，網站不會對新 `XXX` 任務扣點；已建立但逾時的任務將在後續查詢或 Bridge 請求時安全終止並退款。

## 固定工作流程限制

網站只會提供經審核的 `qwen-image-edit-rapid` 模板。Bridge 只允許將 ComfyUI 上傳後的檔名注入節點 `78`，並只接受節點 `102` 的圖片輸出。瀏覽器、任務建立者與 Bridge 以外的任何來源皆不能提供任意工作流程 JSON、節點 ID、模型名稱或提示詞。

## 網路與桌面設定

ComfyUI 應只監聽 `127.0.0.1:8188`。不需要 DDNS、路由器連接埠轉發、CORS、外部 HTTPS 反向代理或公開 ComfyUI API。Bridge 只需對網站建立出站 HTTPS 連線；若電腦、ComfyUI 或 Bridge 未運行，`XXX` 會顯示工作站離線而不建立需收費的工作。
